# 科研龙虾-总览索引

返回入口：[[首页总控台]]

## 定位

科研龙虾的第一阶段定位为：`个人科研工作台`。目标不是先做团队协作平台，而是先把个人研究者最核心的链路做到扎实可用，再通过真实试用反馈不断迭代。

## 当前结论

- 推荐采用 `OpenClaw + 可验证学术证据底座 + 个人研究记忆` 的结构。
- 第一阶段优先解决中国大学生、研究生在 `检索、筛选、精读、引用、沉淀` 五个环节上的高频痛点。
- 第一阶段不追“大而全”，不承诺自动写完整论文，不把团队协作作为主目标。
- OpenClaw 的核心约束更准确地说是 `单信任边界 / 单操作者边界 per Gateway`，不是简单的“单用户”。
- 学术网页、论文 PDF、外部数据库页面都是 `不可信输入`，第一阶段必须按严格工具边界设计。

## 关键判断

- 第一阶段产品愿景可以是全学科通用，但试点覆盖范围必须按数据源现实收窄，优先覆盖 `CS / AI / 生物医学 / DOI-heavy STEM`。
- “合格的科研龙虾”不是会聊天的套壳助手，而是能把 `题目 -> 检索式 -> 文献池 -> 精读卡 -> 证据卡 -> 引用条目` 这条链路做稳的个人科研工具。
- 第一阶段最应该压实的不是写作能力，而是 `真实性、时效性、可追溯性、幻觉抑制、中英双语检索`。
