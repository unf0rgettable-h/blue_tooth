# 科研龙虾-接口与工具契约

返回入口：[[科研龙虾-总览索引]]

## 一、设计原则

科研龙虾 v1 的接口设计应满足：

- 小而稳
- 可审计
- 低歧义
- 强输入约束
- 强输出等级标识

## 二、工具清单

### 1. `query_planner`

#### 输入

- `research_question`
- `discipline_hint`
- `language_preference`

#### 输出

- `zh_terms`
- `en_terms`
- `synonyms`
- `excluded_terms`
- `recommended_sources`
- `suggested_queries`

#### 约束

- 不返回直接学术结论
- 只返回检索策略

### 2. `paper_discovery`

#### 输入

- `topic_id` 或 `query_bundle`
- `source_scope`
- `limit`

#### 输出

- `candidate_papers`
- `source_coverage`
- `query_trace`

#### 约束

- 不直接生成自然语言综述
- 必须保留来源提供者标识

### 3. `citation_verify`

#### 输入

- `paper_candidates`

#### 输出

- `verified_records`
- `needs_manual_confirmation`
- `conflicts`

#### 约束

- 无验证状态不得进入正式引用导出

### 4. `reading_card_generate`

#### 输入

- `paper_id`
- `evidence_level`
- `source_payload`

#### 输出

- `reading_card`
- `warnings`

#### 约束

- `metadata_only` / `abstract_only` 不得输出完整方法/数据/局限

### 5. `workspace_write`

#### 输入

- `object_type`
- `payload`

#### 输出

- `object_id`
- `write_status`

#### 约束

- 所有写入都要带时间戳
- 所有写入要保留来源信息

## 三、OpenClaw 集成边界

### 主 Agent 能做什么

- 判断用户当前属于哪个科研任务
- 调用工具
- 组织结果
- 解释限制和风险

### 主 Agent 不能直接做什么

- 直接把网页搜索结果当正式文献
- 绕过 verifier 生成正式引用
- 在低证据等级下输出高承诺精读卡
- 调用高风险系统工具

## 四、最小请求流程

### 场景 1：从题目出发

1. `query_planner`
2. `paper_discovery`
3. `citation_verify`
4. 返回候选文献池

### 场景 2：从单篇论文出发

1. `citation_verify`
2. `reading_card_generate`
3. `workspace_write`

## 五、错误与告警语义

建议统一 4 类错误：

- `source_unavailable`
- `insufficient_evidence`
- `verification_conflict`
- `manual_confirmation_required`

## 六、v1 响应必须带的元信息

每次关键响应至少带：

- `source_provider`
- `evidence_level`
- `verification_status`
- `manual_review_required`
