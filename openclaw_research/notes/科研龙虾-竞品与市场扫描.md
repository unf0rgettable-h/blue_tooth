# 科研龙虾-竞品与市场扫描

返回入口：[[科研龙虾-总览索引]]

## 一、总体判断

截至 2026-03-30，市面上已经有不少接近“科研龙虾”某个局部能力的产品，但市场仍然是明显碎片化的。

最接近的国际产品包括：

- Elicit
- Consensus
- SciSpace
- scite
- Semantic Scholar
- ResearchRabbit
- Connected Papers
- Perplexity Academic / Deep Research

国内或中文环境下，相关但不完全等价的产品包括：

- CNKI 研学
- NoteExpress
- 秘塔 AI 搜索（学术模式）
- Zotero / EndNote / Mendeley / NoteExpress 等文献管理体系
- 通用大模型产品中的“学术搜索 / 文献问答”玩法

### 关键市场空档

目前没有哪个产品非常明确地同时做好这五件事：

1. 文献发现
2. 单篇深读
3. 证据追踪
4. 引用管理
5. 长期个人科研工作流记忆

这也是科研龙虾最值得切入的空间。

## 二、国际同类产品

### 1. Elicit

#### 它擅长什么

- 文献检索
- 结构化文献综述
- 研究报告
- 论文筛选与信息提取

#### 它值得借鉴的地方

- 强调 evidence synthesis，而不是单纯聊天
- 对研究综述类任务支持较强

#### 它的缺口

- 更偏文献综述与研究提取，不像一个完整的“个人科研操作系统”
- 长期个人研究记忆与工作流沉淀感不够强

#### 官方入口

- https://elicit.com/
- https://elicit.com/solutions/search
- https://elicit.com/solutions/systematic-review

### 2. Consensus

#### 它擅长什么

- 学术搜索
- 快速问答
- 研究结论聚合
- Deep Search / 文献回顾类生成

#### 它值得借鉴的地方

- 把“学术问答”做得很接近用户直觉
- 更强调用论文回答问题，而不是泛网页答案

#### 它的缺口

- 适合快速找方向，但长期项目化工作流和证据审计不够深

#### 官方入口

- https://help.consensus.app/en/articles/9922673-how-consensus-works
- https://consensus.app/home/blog/deep-search/

### 3. SciSpace

#### 它擅长什么

- 文献搜索
- PDF 对话
- 文献综述工作区
- 引用生成

#### 它值得借鉴的地方

- 对“阅读体验”和“研究工作台感觉”做得比较完整
- 多语言和 PDF 交互体验较强

#### 它的缺口

- 检索透明度和证据严谨性不够突出
- 更像一套广功能工具，而不完全是 rigor-first 研究系统

#### 官方入口

- https://scispace.com/search
- https://scispace.com/resources/scispace-literature-review-workspace/

### 4. scite

#### 它擅长什么

- Smart Citations
- 引文上下文
- 支持 / 反对 / 提及关系
- 参考文献检查

#### 它值得借鉴的地方

- 在“证据追踪”这一块很强
- 非常适合做科研龙虾里的证据层参照物

#### 它的缺口

- 不是完整科研工作流工具
- 发现、深读、长期个人记忆都不是它最强的部分

#### 官方入口

- https://scite.ai/
- https://scite.ai/reports/scite-a-smart-citation-index-keppkgL5

### 5. ResearchRabbit / Connected Papers / Semantic Scholar

#### 它们擅长什么

- ResearchRabbit：发现与追踪
- Connected Papers：种子论文周边图谱
- Semantic Scholar：免费搜索与基础研究入口

#### 借鉴价值

- ResearchRabbit 和 Connected Papers 很适合启发“发现层”
- Semantic Scholar 很适合作为免费基座与用户熟悉入口

#### 缺口

- 都没有完整打通“发现 -> 精读 -> 证据 -> 引用 -> 长期记忆”全链路

#### 官方入口

- https://www.researchrabbit.ai/
- https://www.connectedpapers.com/about
- https://www.semanticscholar.org/product

## 三、中文 / 国内相关产品与工具

### 1. CNKI 研学

#### 它擅长什么

- 与中国学术数据库深度结合
- 文献检索、阅读、笔记、管理、学习场景结合

#### 它值得借鉴的地方

- 很贴近中国学生的真实数据库环境
- 把“检索 + 阅读 + 笔记”放在同一条链路上

#### 它的缺口

- 对跨中英数据库、多源证据归一、开放学术世界的整合不够强
- 很难成为一个真正跨源的个人科研中枢

#### 参考入口

- 高校图书馆介绍页：
  https://tsg.hfnu.edu.cn/info/1022/9911.htm

### 2. NoteExpress

#### 它擅长什么

- 文献管理
- 引用格式化
- 中国学术环境中的论文写作配套

#### 它值得借鉴的地方

- 非常贴近中国论文引用与管理习惯
- 在“参考文献管理”这一传统环节仍然强势

#### 它的缺口

- 它是文献管理器，不是 AI 科研助手
- 检索、深读、证据总结、长期研究工作流不是它的核心

#### 官方入口

- http://www.inoteexpress.com/

### 3. 秘塔 AI 搜索（学术模式）

#### 它擅长什么

- 学术搜索模式
- 结构化答案
- 引用源展示
- 中英数据库可配置

#### 它值得借鉴的地方

- 更贴近中文用户的搜索习惯
- 结构化答案和学术模式对学生很友好

#### 它的缺口

- 依然更偏“搜索 + 整理”，不是完整科研工作流系统
- 个人长期研究记忆、严谨证据层、引用核验链路不够明确

#### 官方入口

- https://metaso.cn/

### 4. 通用大模型产品中的学术玩法

如 Kimi、豆包、通义、腾讯元宝等，在实际学生使用中会承担：

- 快速问答
- 论文摘要理解
- 写作润色
- 初步搜索

#### 它们的共同问题

- 通用性强，但研究流程不够专门化
- 来源、引用、版本、撤稿、证据链很难天然做严
- 容易在科研场景下“看起来很懂”

## 四、对科研龙虾最有价值的借鉴

### 应借鉴

- Elicit 的结构化综述能力
- Consensus 的“用论文回答问题”体验
- SciSpace 的阅读工作台感
- scite 的证据追踪能力
- ResearchRabbit / Connected Papers 的发现层能力
- NoteExpress / CNKI 研学对中国科研现实的贴近度

### 不应照搬

- 只做学术聊天问答
- 只做 PDF 对话
- 只做文献管理
- 只做图谱探索

科研龙虾第一阶段真正该做的是：把这些能力背后的“最有价值部分”整合成一个更完整、更可验证的个人科研工作台。

## 五、对科研龙虾的启发

### 1. 市场已经证明“有需求”

说明这条路不是伪需求。

### 2. 市场仍然碎片化

说明科研龙虾仍然有切入口。

### 3. 中国语境仍然缺一款真正贴地气的个人科研工作台

尤其是在：

- 中英双语检索
- 中国数据库现实约束
- 引用规范
- 个人长期研究记忆

这些维度上，仍然存在明显空档。
