# SurvLink v1.3.0 Release Notes

## Highlights

- Split the app into two bottom-navigation pages:
  - `蓝牙`
  - `数据`
- Kept Bluetooth setup and connection actions on the Bluetooth page
- Moved receive/import/export/file actions to the Data page
- Made the imported-file panel permanently visible with an empty state when no file exists yet
- Re-locked brand/model/device selection when a current session exists
- Stopped silently wiping the current session when attempting to switch to a different device
- Added model-aware import behavior:
  - TS09 stays on the active batch-import path
  - TS60 now surfaces guidance/limitations instead of assuming the TS09 path always works

## Reliability Notes

- The native code path still uses blocking Bluetooth reads for live receive
- Import metadata is now persisted independently from the current live session
- `lintDebug`, `assembleDebugAndroidTest`, and `assembleRelease` passed in the current WSL environment

## Known Limits

- TS60 batch import is not declared fully supported in this release; the app now exposes guidance instead of treating it as identical to the TS09 path
- No connected Android device/emulator was available in this environment for connected instrumentation tests or live field validation
