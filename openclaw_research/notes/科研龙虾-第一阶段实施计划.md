# 科研龙虾第一阶段实施计划

> **For agentic workers:** REQUIRED: Use `subagent-driven-development` for execution. Steps use checkbox syntax for tracking.

**Goal:** 在 `118.145.222.97` 上部署一个基于 OpenClaw 的个人版科研龙虾 v1，先把“研究问题转检索策略 -> 候选论文池 -> 引用验证 -> OA/PDF 精读卡”这条主链路打通。

**Architecture:** 采用 `OpenClaw Gateway + 受限科研工具层 + 可验证学术证据底座 + 个人研究工作区` 的结构。第一阶段按单信任边界部署，工具默认最小权限，学术内容视为不可信输入，先做可验证的窄主链路，不追求全学科与全数据库能力。

**Tech Stack:** OpenClaw stable, Node.js, Python 3.11+, Crossref/OpenAlex/arXiv/PubMed/Europe PMC/Unpaywall APIs, local metadata store, local evidence cache, Obsidian vault for planning and ops records.

## 风险清单

- 中国数据库接入更像合规问题，不只是工程问题。
- OpenClaw 默认能力过强，必须严格做工具白名单。
- 如果第一轮试点学科太散，会把“覆盖不足”误判成“产品差”。
- 如果没有证据等级，精读卡会天然过度自信。

## 执行顺序建议

1. 先做 `部署与安全基线`
2. 再做 `query planner + candidate pool + citation verifier`
3. 然后做 `reading cards`
4. 最后再做 `OpenClaw 工作流集成`
5. 试用只在主链路稳定后开始
