# 科研龙虾-技术架构蓝图

返回入口：[[科研龙虾-总览索引]]

## 一、v1 总体架构

科研龙虾 v1 建议采用 4 层结构：

1. `OpenClaw Gateway 层`
2. `科研工作流编排层`
3. `学术证据服务层`
4. `本地研究工作区与存储层`

这 4 层的核心原则是：

- OpenClaw 负责会话与调用，不负责学术事实判定
- 工作流层负责组织步骤，不负责伪造证据
- 学术证据服务层负责检索、验证、归一、精读闸门
- 存储层负责持久化研究对象与审计痕迹

## 二、组件划分

### 1. Gateway 组件

#### `research-lobster-gateway`

职责：

- 接受用户请求
- 识别任务类型
- 调用受限工具
- 返回可追溯回答

约束：

- 只运行在单信任边界内
- loopback 绑定
- 不开放 shell / sudo / 浏览器自动化

### 2. 编排组件

#### `query-planner`

职责：

- 将题目转成中英双语检索策略
- 输出数据库路由建议

#### `candidate-pool-orchestrator`

职责：

- 驱动多源检索
- 聚合候选论文池
- 调用 verifier / ranker

#### `reading-card-orchestrator`

职责：

- 根据证据等级决定能否生成完整精读卡
- 将回答与卡片沉淀进工作区

### 3. 学术证据服务

#### `metadata-resolver`

职责：

- Crossref / PubMed / PMC / OpenAlex / arXiv 等解析
- 返回统一 schema

#### `dedupe-linker`

职责：

- DOI / PMID / PMCID 级去重
- 预印本 / 正式版 / 更新关系链接

#### `citation-verifier`

职责：

- 正式引用路径验证
- 非 ID 路径进入人工确认状态

#### `evidence-gate`

职责：

- 判断当前文献是 metadata-only / abstract-only / OA-fulltext / user-uploaded-pdf
- 决定允许生成何种强度的输出

#### `reading-card-generator`

职责：

- 在闸门允许范围内生成精读卡
- 强制附带证据等级与来源

### 4. 存储层

#### `workspace-store`

建议至少包含：

- 主题卡集合
- 论文卡集合
- 证据卡集合
- 引用条目集合
- 任务日志

#### `cache-store`

建议缓存：

- 元数据查询结果
- 候选池中间结果
- 可合法缓存的开放摘要 / 开放全文索引

## 三、部署形态

### v1 推荐形态

- 一台 Ubuntu 服务器
- 一套 OpenClaw Gateway
- 一套科研龙虾服务目录
- 单用户 / 单信任边界使用

### 不建议的形态

- 多租户同机共享 Gateway
- 一个 Gateway 上同时跑高风险自动化工具与研究助手
- 直接把学校数据库抓取链路做进主服务

## 四、服务之间的调用关系

用户请求的典型路径：

1. 用户输入研究问题
2. OpenClaw 调用 `query-planner`
3. 编排层调用 `candidate-pool-orchestrator`
4. 候选池调用 `metadata-resolver` + `dedupe-linker` + `citation-verifier`
5. 若用户要求精读，则进入 `evidence-gate`
6. 闸门放行后再调用 `reading-card-generator`
7. 结果与卡片写入 `workspace-store`

## 五、v1 需要刻意避免的架构错误

### 1. 让 LLM 直接面对原始搜索结果后自由发挥

这会导致：

- 引用虚构
- 证据等级混乱
- 摘要级信息过度外推

### 2. 把工作流状态和学术事实混在同一层

应该把：

- 用户任务状态
- 学术元数据
- 证据判断
- 精读输出

拆开建模。

### 3. 让 OpenClaw 直接接触高 blast-radius 工具

研究型助手的默认边界应偏保守，而不是全能。

## 六、v1 最小落地形态

如果要尽快开始实现，建议第一版只落这条最小主链：

- `query-planner`
- `candidate-pool-orchestrator`
- `metadata-resolver`
- `citation-verifier`
- `evidence-gate`
- `reading-card-generator`
- `workspace-store`
