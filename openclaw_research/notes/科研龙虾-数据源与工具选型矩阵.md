# 科研龙虾-数据源与工具选型矩阵

返回入口：[[科研龙虾-总览索引]]

## 一、选型原则

第一阶段选型遵循 5 条原则：

- `权威元数据优先`
- `聚合源只做补充，不做唯一真相`
- `能验证胜过能搜更多`
- `先支持开放与可核验内容，再谈深接封闭库`
- `先把单用户主链路做稳，再扩工具面`

## 二、数据源矩阵

| 数据源 | 第一阶段角色 | 用来解决什么 | 优点 | 风险 / 限制 | 建议优先级 |
|---|---|---|---|---|---|
| Crossref | 权威元数据骨架 | DOI、题录、期刊、更新时间、Crossmark | 权威、标准化强 | 不覆盖所有中文/非 DOI 来源 | P0 |
| OpenAlex | 广度补充 | 扩召回、补图谱、补 open access 位置 | 覆盖广、开发友好 | 不能当唯一真相源 | P0 |
| arXiv | 预印本入口 | CS/AI 等方向预印本发现 | 对 AI/CS 很关键 | 需和正式版关系处理 | P0 |
| PubMed | 医学检索骨架 | 生物医学检索、ID 与主题支持 | 医学领域强 | 只覆盖部分学科 | P0 |
| Europe PMC | 医学生态补充 | 引文、PMC 关系、开放内容 | 医学相关增强 | 非医学覆盖有限 | P0 |
| Unpaywall | OA 补充 | 找开放全文入口 | 对 OA 获取很有用 | 不是元数据权威源 | P0 |
| DataCite | 补充权威层 | 数据集、部分 DOI 元数据 | 对非传统研究对象有用 | 第一阶段可后接 | P1 |
| Semantic Scholar | 广度与引文补充 | 引文图谱、相关论文 | 免费基座强 | 官方依赖边界需谨慎控制 | P1 |
| DOAJ | OA 期刊侧校验 | OA 期刊判断 | 辅助质量判断 | 不是核心检索入口 | P1 |
| CNKI | 中国语境入口 | 数据库路由、元数据 / 用户文件后处理 | 中文学术现实不可绕开 | 合规、题录摘要边界、授权限制 | P1 |
| 万方 | 中国语境入口 | 同上 | 真实场景重要 | 同上 | P1 |
| 维普 | 中国语境入口 | 同上 | 真实场景重要 | 同上 | P1 |

## 三、工具矩阵

| 工具 / 组件 | 第一阶段用途 | 是否直接采用 | 原因 |
|---|---|---|---|
| OpenClaw | 主 Gateway / 编排层 | 是 | 适合作为单信任边界、长驻型科研助手入口 |
| Firecrawl | 深网页检索 / 市场与文档研究 | 是，但不进入终态核心链路 | 适合调研与开放网页内容，不适合作为受许可学术库主入口 |
| Tavily | 快速对比检索 | 是，作为辅助研究工具 | 适合外部信息扫描与比较 |
| Obsidian | 方案、运维、研究记录 | 是 | 适合长期沉淀设计、运维记录和迭代轨迹 |
| PaperQA2 | 技术参考 | 参考，不直接作为第一阶段核心 | 路线很值得学，但要结合中国语境和 OpenClaw 重新设计 |
| scite | 竞品参考 / 证据追踪标杆 | 参考 | 引文上下文能力值得学习 |
| Elicit / Consensus / SciSpace | 竞品参考 | 参考 | 用来定义差异化，而不是直接依赖 |

## 四、模型与能力选型原则

### 1. Query Planner

建议：强模型优先  
原因：研究问题转检索策略比普通问答更依赖术语展开和双语映射。

### 2. Candidate Pool Assembly

建议：尽量规则化、服务化  
原因：这是证据底座，不应过度依赖 LLM“猜测”。

### 3. Citation Verifier

建议：规则 + 权威 API  
原因：引用验证应尽量少用生成模型。

### 4. Reading Cards

建议：LLM 参与，但必须受证据等级闸门限制  
原因：精读卡是最容易产生“看起来很懂”的地方。

## 五、第一阶段明确不选

- 不选“全网网页搜索结果”作为正式学术引用真相源
- 不选“单一聚合学术平台”作为唯一真相源
- 不选“自动登录高校数据库 + 全文抓取”作为第一阶段主策略
- 不选“在共享 Gateway 上做复杂团队权限系统”作为团队版演进路径

## 六、推荐的第一阶段最小组合

### 检索与验证主链路

- Crossref
- OpenAlex
- arXiv
- PubMed
- Europe PMC
- Unpaywall

### 编排与承载

- OpenClaw
- 本地受限工作区
- 本地可缓存元数据层

### 研究与运维辅助

- Obsidian
- Firecrawl
- Tavily

## 七、后续可扩展方向

- DataCite 更深接入
- Semantic Scholar 更深引文能力
- 中国数据库更稳妥的授权内接入策略
- 学科专精适配层
- 团队版多 Gateway 架构
